# 模型类产品命名规范

## 命名规则

### 标准格式

**模型_序号_类别_产品名称.md**

---

## 命名规则详解

### 1. 前缀固定部分

```
模型_
```
- 必须以"模型_"开头
- 标识产品属于模型类
- 便于文件分类和搜索

### 2. 序号部分

```
001, 002, 003, ...
```
- 使用三位数字编码
- 从001开始递增
- 便于排序和管理
- 004-009已使用

### 3. 类别部分

```
类别_名称
```
- 使用"类别_"前缀
- 标识产品所属的大类
- 便于分类查找

| 类别代码 | 中文名称 | 说明 |
|----------|----------|------|

| 地形模型 | Terrain | 地形、地质类 |

### 4. 产品名称部分

```
产品名称
```
- 使用英文单词
- 单词之间用下划线连接
- 简洁准确描述产品

## 新增产品命名流程

### Step 1: 确定序号
```
查看现有产品，按顺序分配下一个序号
最后序号：009 → 下一个序号：010
```

### Step 2: 选择类别
```
根据产品特征选择合适类别：
- 沙盘模型：建筑、场景、装饰类
- HO模型：1:87比例的模型
- 人物模型：人形角色类
- 植物模型：植物、草地、树木类
- 其他：根据实际情况选择
```

### Step 3: 确定产品名称
```
将产品名称转换为英文单词：
- 产品名称：微缩建筑模型 →建筑模型：miniature_building_models
- 产品名称：迷你人物套装 →人物模型：mini_figure_set
- 产品名称：仿真草地 →植物模型：artificial_grass
```

### Step 4: 组合完整文件名
```
格式：模型_序号_类别_产品名称.md
示例：模型_010_沙盘_建筑模型.md
```

## 重命名操作指南

### 现有文件重命名
```
1. 找到需要重命名的文件
2. 按照新规则分解为：序号+类别+产品名称
3. 创建新文件名
4. 将内容复制到新文件
5. 删除原文件
```

### 新文件创建流程
```
1. 确定序号（检查最新序号）
2. 选择合适类别
3. 确定产品名称英文
4. 创建文件名
5. 按模板创建内容
```

---

## 注意事项

1. **序号连续**：不要跳号，按顺序使用
2. **类别准确**：根据产品特征选择最合适的类别
3. **名称简洁**：英文单词不要过长，2-3个单词为佳
4. **格式统一**：严格使用下划线连接，不要有空格
5. **大写不敏感**：文件名不区分大小写，建议使用小写
