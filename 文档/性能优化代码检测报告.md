## 二、CSS 过渡 & 动画优化（可能直接导致回闪）

### 2.1 全局元素过渡 [themes-light-dark.css](file:///e:/Product%20Description/product-page-generator/public/css/themes-light-dark.css#L9-L15)

```css
*,
*::before,
*::after {
  transition: background-color var(--transition-slow),
              color var(--transition-slow),
              border-color var(--transition-slow),
              box-shadow var(--transition-slow);
}
```

> **问题**: 对所有元素（包括 input 输入框）启用了 `background-color`、`color`、`border-color`、`box-shadow` 过渡。当输入框状态变化时，过渡效果可能导致视觉回闪。

**禁用方案**: 移除全局 `*` 过渡，仅对需要主题切换的元素添加过渡。

---

### 2.2 no-transition 防闪策略 [themes-light-dark.css](file:///e:/Product%20Description/product-page-generator/public/css/themes-light-dark.css#L17-L21)

```css
.no-transition *,
.no-transition *::before,
.no-transition *::after {
  transition: none !important;
}
```

> **问题**: `no-transition` class 的添加/移除时机可能造成过渡闪烁竞态。

---

### 2.3 主题管理器防闪 [themeManager.js](file:///e:/Product%20Description/product-page-generator/public/js/services/themeManager.js#L19-L24)

```javascript
init() {
    this.applyTheme(this.currentTheme);
    this.watchSystemTheme();
    // 添加首次加载标记，避免过渡动画
    document.documentElement.classList.add('no-transition');
    setTimeout(() => {
      document.documentElement.classList.remove('no-transition');
    }, 100);

```

> **问题**: 
> 1. 固定 100ms 延迟不可靠。
> 2. `no-transition` class 的添加/移除可能与其他 DOM 操作产生竞态。
> 3. 重复初始化（多次调用 `new ThemeManager()`）会重置 `no-transition`。

**禁用方案**: 移除 `no-transition` 机制，改用 CSS `@starting-style` 或初始状态跳过过渡。

## 三、缓存策略（内存泄漏风险）

### 3.1 页面加载缓存 [page-loader.js](file:///e:/Product%20Description/product-page-generator/public/core/page-loader.js#L3-L60)

```javascript
class PageLoader {
    constructor() {
        this._cache = new Map();        // HTML 内容缓存 - 无上限
        this._loadedScripts = new Set(); // 脚本加载去重
        this._loadedCSS = new Set();     // CSS 加载去重
    }
    // ...
    loadHTML(url) {
        if (this._cache.has(url)) return this._cache.get(url); // 缓存命中
        // ...
        this._cache.set(url, html); // 缓存存储 - 无大小限制
    }
}
```

> **问题**: `_cache` Map 无大小上限，加载的页面越多，内存占用越大。单例模式不会调用 `clearCache()`。

**禁用方案**: 
1. 移除 `_cache`，每次重新 fetch（或设置最大条目数）
2. 移除 `_loadedScripts` / `_loadedCSS` 去重（改为条件加载）

---

### 3.2 路由脚本去重 [router.js](file:///e:/Product%20Description/product-page-generator/public/core/router.js#L62-L67)

```javascript
if (!this._loadedScripts.has(scriptPath)) {
    this._loadedScripts.add(scriptPath);
    await this._loadScript(scriptPath);
}
```

> **问题**: `_loadedScripts` Set 无上限，但影响小（最多等于路由数）。

### 3.3 翻译缓存 [translation.js](file:///e:/Product%20Description/product-page-generator/public/js/modules/translation.js#L1-L56)

```javascript
class TranslationManager {
    constructor() {
        this.cache = new Map(); // 内存缓存 - 无上限
        this._loadCache();       // 从 localStorage 加载所有历史缓存
    }

    _loadCache() {
        // 遍历所有 localStorage key，加载 trans_cache_ 前缀的数据
        const cacheKeys = keys.filter(k => k.startsWith('trans_cache_'));
        for (const key of cacheKeys) {
            this.cache.set(mapKey, data.product); // 全部加载到内存
        }
    }
}
```

> **问题**:
> 1. 初始化时从 localStorage 加载全部翻译缓存到内存 Map，无上限。
> 2. `saveToStorage` 永久存储，localStorage 可能满。
> 3. 缓存键包含产品 ID 和语言，产品越多缓存越大。

**禁用方案**: 
1. 移除 `_loadCache()` 初始化加载
2. 移除 `setToMemory()` / `getFromMemory()` 
3. 保留 `getFromStorage()` / `saveToStorage()` 以便后续逐步开启

## 四、防抖 & 节流 [utils.js](file:///e:/Product%20Description/product-page-generator/public/js/services/utils.js#L41-L64)

### 4.1 debounce

```javascript
export const debounce = (func, wait) => {
    let timeout;
    return function executedFunction(...args) {
        const later = () => {
            clearTimeout(timeout);
            func(...args);
        };
        clearTimeout(timeout);
        timeout = setTimeout(later, wait);
    };
};
```

> **问题**: 
> 1. 传入的函数被延迟执行，可能导致输入框内容与状态不同步。
> 2. `clearTimeout` 后 `func` 仍在 `later` 闭包中，如果 `func` 持有 DOM 引用，可能延迟 GC。

**禁用方案**: 用直接调用替换防抖。

---

### 4.2 throttle

```javascript
export const throttle = (func, limit) => {
    let inThrottle;
    return function(...args) {
        if (!inThrottle) {
            func.apply(this, args);
            inThrottle = true;
            setTimeout(() => inThrottle = false, limit);
        }
    };
};
```

> **问题**: 第一次调用后的 `limit` 时间内后续调用被丢弃。

**禁用方案**: 用直接调用替换节流。

### 6.1 AI Chat - 全局 mousemove/mouseup 未清理 [ai-chat.js](file:///e:/Product%20Description/product-page-generator/public/components/ai-chat/ai-chat.js#L85-L108)

```javascript

    if (_isDragging) { ... }
    if (_isResizing) { ... }

document.addEventListener('mouseup', () => {      // 匿名函数 - 无法移除
```

## 七、状态管理订阅 [appState.js](file:///e:/Product%20Description/product-page-generator/public/js/stores/appState.js)

### 7.1 订阅者模式

```javascript
subscribe(listener) {
    this.listeners.push(listener);
    return () => {
        this.listeners = this.listeners.filter(l => l !== listener);
    };
}
```

> **问题**: 
> 1. 订阅者需要手动调用 unsubscribe 返回值。
> 2. `this.listeners` 数组的 filter 操作在每次取消时，O(n) 复杂度。
> 3. 如果不再使用但未取消订阅，listener 持有闭包引用，阻止 GC。
