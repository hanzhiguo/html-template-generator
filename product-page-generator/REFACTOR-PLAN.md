---

## 🎯 拆分原则

| 原则 | 说明 |
|------|------|
| **单一职责** | 每个模块只做一件事 |
| **低耦合** | 模块间通过事件/回调通信，不直接调用 |
| **高复用** | 工具函数、API调用独立抽取 |
| **边界清晰** | 每个模块有明确的输入/输出 |

## 🔌 模块间通信设计

```
┌──────────────────────────────────────────────────┐
│                    app.js (主入口)                 │
│  创建 App 实例，注册所有模块                       │
└────────────┬─────────────────────────────────────┘
             │ 事件总线 / 直接引用
    ┌────────┼────────┬────────┬────────┬─────────┐
    ▼        ▼        ▼        ▼        ▼         ▼
  Auth   Products  Preview  Agent   Settings  Attachments
    │        │        │        │        │         │
    └────────┴────────┴────────┴────────┴─────────┘
             │ 共享服务层
    ┌────────┼────────┬────────┬────────┐
    ▼        ▼        ▼        ▼        ▼
  storage  utils   appState   api/*   themeManager
```

### 通信方式

1. **直接引用**：模块通过 `this.xxx` 访问其他模块方法
2. **事件总线**：跨模块通知使用 CustomEvent
3. **共享状态**：通过 `appState` 统一管理

## ⚠️ 风险控制

1. **每阶段独立测试**：完成一个阶段立即测试，不累积问题
2. **保留备份**：`app.js.bak` 已存在，每阶段前再备份
3. **渐进式替换**：新模块先与旧代码并存，验证后再删除旧代码
4. **HTML 不改动**：阶段 1-9 不改 index.html，阶段 10 统一更新
