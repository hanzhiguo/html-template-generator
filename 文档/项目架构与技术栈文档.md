# Novel Board AI 项目架构与技术栈文档

---

## 一、项目概述

**Novel Board AI** 是一款 AI 驱动的小说转视频创作平台，通过集成大语言模型（LLM）和图像/视频生成技术，将小说文本自动转化为情节分镜、角色设计、场景描述及最终视频短片。

- **版本**: 0.9.95
- **定位**: 自媒体博主、编剧、小说爱好者的创作工具
- **核心价值**: 降低内容创作门槛，提升文字到视觉作品的转化效率

## 四、核心架构设计

### 4.1 状态管理架构

采用 **Context + Reducer** 模式，实现轻量级全局状态管理：

```tsx
// stores/AppContext.tsx
const AppContext = createContext<AppContextType | undefined>(undefined);

export const AppProvider: React.FC<AppProviderProps> = ({ children }) => {
  const [state, dispatch] = useReducer(appReducer, initialAppState);
  return (
    <AppContext.Provider value={{ state, dispatch }}>
      {children}
    </AppContext.Provider>
  );
};

export const useAppContext = () => useContext(AppContext);
```

  const maxPolls = 600;
  for (let i = 0; i < maxPolls; i++) {
    const status = await fetch(`/v1/api/result?id=${taskId}`);
    if (status.status === 'succeeded') {
      return status.results;
    }
    await sleep(3000);
  }
  throw new Error('Task timeout');
};

## 十四、最佳实践总结

### 14.1 代码组织

1. **模块化**: 按功能划分模块，每个模块独立维护
2. **类型安全**: 严格使用 TypeScript 类型定义
3. **单一职责**: 每个文件控制在 2000 行以内
4. **命名规范**: 使用语义化的文件和函数命名

### 14.2 状态管理

1. **最小状态**: 只存储必要的状态
2. **派生状态**: 使用 useMemo 计算派生状态
3. **状态提升**: 共享状态提升到共同父组件

### 14.3 异步处理

1. **任务队列**: 长时间任务使用队列管理
2. **错误重试**: 网络请求实现重试机制
3. **超时处理**: 设置合理的超时时间

### 14.4 性能优化

1. **按需加载**: 路由级和组件级懒加载
2. **缓存策略**: 合理使用缓存减少重复计算
3. **防抖节流**: 频繁操作使用防抖节流
